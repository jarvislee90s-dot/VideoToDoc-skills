# 思维导图渲染形态重构设计

## 背景

当前 `video-to-slides` 生成的思维导图使用 Mermaid `mindmap` 语法，渲染后呈**中央标题向四周圆形散射**结构。对于章节多、节点多的长视频，这种布局会导致：

- 阅读顺序混乱，难以跟随章节逻辑；
- 节点互相重叠，三级分支难以辨识；
- 整体图像过宽或过窄，不适合作为讲义配图。

本设计将思维导图改为**目录树状结构**，并根据节点数量自动选择根节点位置和分栏策略。

## 设计目标

1. **章节逻辑优先**：一级节点必须是章节标题，按目录顺序排列。
2. **向右侧展开**：每个节点向右延伸，不超过 3 个层级。
3. **自动分栏**：章节多时自动分栏，避免单栏过长。
4. **根节点自适应**：
   - 多栏时根节点置于顶部居中；
   - 单栏时根节点置于左侧。
5. **节点清晰可辨**：
   - 每个节点加框；
   - 按层级使用颜色和字号区分；
   - 节点之间保留最小间距，避免重叠；
   - 跨栏连接线使用虚线，不穿过其它节点。
6. **仍输出为单张图片**：最终产物仍是一张 PNG，可直接插入 Word/Markdown/飞书。

## 总体方案

### 渲染后端选择

现有两种渲染路径：

- **Mermaid `mindmap`**：默认使用，但无法关闭圆形散射，布局不可控。
- **Python fallback 渲染器**：当 `mmdc` 不可用时启用，使用 Pillow 绘制横向树状图。

本设计将 **Python fallback 渲染器提升为默认渲染器**，并扩展其能力以支持分栏和自适应根节点位置。Mermaid 路径保留为可选项（通过 `--mermaid` 参数显式开启），便于未来需要 Mermaid 源文件的场景。

### 数据结构

思维导图数据源仍来自 Agent 生成的 `.mmd` 文件中的层级文本结构。Python 渲染器解析后得到如下树：

```json
{
  "root": "Git+GitHub",
  "children": [
    {
      "title": "基础概念",
      "children": ["Git", "GitHub", "仓库类型"]
    },
    ...
  ]
}
```

当前 `.mmd` 使用 Mermaid `mindmap` 语法；渲染器需要新增一个轻量解析器，将其转换为上述树结构。

### 布局算法

1. **确定单栏容量**
   - 设定单栏最大高度 `MAX_COL_H`（默认 520 px）。
   - 按章节顺序累加每个章节的子树高度，超过阈值则开启新栏。

2. **计算每栏尺寸**
   - 章节标题框宽度固定 `CHAPTER_W = 130 px`。
   - 叶子节点框宽度固定 `LEAF_W = 145 px`。
   - 同章节内叶子节点垂直堆叠，间距 `LEAF_GAP = 12 px`。
   - 章节之间间距 `CHAPTER_GAP = 40 px`。

3. **叶子节点最小间距**
   - 在自底向上布局时，相邻叶子节点之间强制保留 `MIN_LEAF_GAP = 14 px`。
   - 章节框垂直居中于其子树簇。
   - 章节框之间再保留 `CHAPTER_GAP`。

4. **多栏根节点位置**
   - 若栏数 `> 1`：
     - 根节点 `Git+GitHub` 置于 SVG 顶部居中；
     - 从根节点向下拉短虚线至顶部横梁；
     - 顶部横梁以虚线连接每一栏的入口；
     - 每栏内部仍按左→右树状展开。
   - 若栏数 `= 1`：
     - 根节点置于左侧；
     - 向右依次展开章节和叶子节点。

5. **曲线连接**
   - 根到栏入口：虚线。
   - 栏入口到章节：平滑 Bézier 曲线。
   - 章节到叶子：平滑 Bézier 曲线。
   - 所有线条不穿过节点框。

### 样式规范

| 层级 | 背景 | 边框 | 字号 | 字重 |
|------|------|------|------|------|
| 根节点 | `#4caf50` | 无 | 14 px | bold |
| 一级章节 | `#dbeafe` | `#1c60af` 1.5 px | 12 px | 600 |
| 二级/三级叶子 | `#f8fafc` | `#cbd5e1` 1 px | 10 px | normal |

- 根节点文字白色，其余黑色。
- 虚线：绿色，dasharray `8 5`（主梁）或 `6 4`（下垂线）。
- 曲线：浅蓝灰 `#aac2e6`。

### 输出格式

- 默认输出 PNG（当前 `render_mindmap.py` 的输出）。
- 不再生成 `mindmap` 格式的 `.mmd`；改为生成一个中间 JSON 或直接渲染。
- 如需保留可编辑源文件，可生成 Mermaid `flowchart LR` 作为副产物。

## 边界情况处理

1. **章节数量 ≤ 4**：大概率单栏，根节点在左，避免顶部留白。
2. **章节数量 > 4 但子节点很少**：按高度分栏，可能分成 2 栏而非 3 栏。
3. **叶子文本过长**：
   - 渲染器内自动换行，最多 2 行；
   - 超过 2 行时截断并加 `…`；
   - 完整文本通过 `title` 属性保留，鼠标 hover 可查看。
4. **整体宽度超过 3000 px**：
   - 限制最大栏数（默认 4 栏）；
   - 仍超限则允许图像宽度突破，或提示 Agent 精简节点。
5. **只有根节点没有子节点**：输出仅含根节点的最小图片。

## 对现有流程的影响

- `video-to-slides` 的 `render_mindmap.py` 是主要修改点。
- `.mmd` 文件格式可保留，但默认不再作为最终渲染入口；渲染器内部解析 `.mmd` 并绘制。
- 飞书/Word 产物无需改动，直接引用 PNG。
- 新增依赖：仅需 Pillow（已存在）。

## 测试计划

1. 用 5 分钟短视频（节点少）验证单栏根节点在左。
2. 用 40 分钟中长视频（节点多）验证多栏根节点在顶、无重叠。
3. 用超长文本节点验证自动换行/截断。
4. 对比 Mermaid 旧输出与 Python 新输出，确保内容一致。
